# Resume Positioning

RingTuner is strongest as a systems simulation project with RL-ready structure, not yet as a completed RL training project.

Possible resume bullet:

```text
Built RingTuner, a C-based deterministic NIC interrupt coalescing simulator with RX ring modeling, interrupt moderation baselines, latency/drop/CPU-cost metrics, mixed-workload threshold tuning, scenario suites, a dependency-free adaptive bandit controller, and an RL-ready step API for future PufferLib Ocean integration.
```

Honest framing:

- C systems simulator.
- Interrupt coalescing tradeoff model.
- Deterministic benchmark harness.
- Offline tuning baseline.
- Dependency-free adaptive bandit baseline.
- RL-ready interface.
- Future PufferLib Ocean candidate.

Avoid claiming:

- Hardware accuracy.
- Completed RL training.
- Production NIC behavior.
- Validated real-world performance.

Use the docs sample result as the public table. It compares realistic baselines and clearly labels the oracle baseline.
